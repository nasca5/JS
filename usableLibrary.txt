1. Swiper : 캐러셀같이 이미지 슬라이드 되는 것들을 좀 예쁘고 쉽게 만들고 싶을 때 쓰면 좋다. 
호환성도 좋고 이미지 lazy loading도 되고 터치/드래그도 지원한다.

2. Chart.js : 웹페이지에 차트 만들고 싶을 때 쓰면 좋다.
실제 서비스를 관리할 때, admin 페이지 만들 때도 많은데, 그러 때 통계를 이쁘게 보여줄 수 있어 유용하다.

3. Animated On Scroll : 스크롤 내리면 요소가 서서히 등장하는 애니메이션을 만들고 싶을 때 쓰면 좋다.
직접 만들어도 되지만, 효율적으로 할 수 있다면 하는 것이 좋으니깐.

4. EmailJS : 원래 이메일 전송은 서버가 하는 일이지만, Gmail 이런 곳의 서버를 잠깐 빌리면 js만으로도 이메일 전송이 가능합니다.
유저가 내 이메일 계정으로 이메일 전송도 가능하고 내 이메일 계정으로 남에게 이메일 전송도 가능하다.
사이트 가서 가입하고 계정 만들고 하면 할 수 있다.

5. Lodash : array. object, 문자, 숫자 자료를 다루기 편해지는 기본 함수들을 제공해준다.
예를 들어서 array 자료에 있는 특정 문자를 제거하고 싶으면 그냥 _.pull(array자료, 'a'); 이런 식으로 쓰면 된다.
용량도 얼마 안 돼서 어느 정도 실력이 쌓인 중급자 정도면 효율성을 높이는 데 괜찮다.

6. React or Vue : 페이지가 너무 많아서 UI 재활용이 자주 필요한 사이트나 모바일 앱처럼 페이지 이동없이 동작하는 Single Page Application을 만들 때 유용한 라이브러리이다.
워낙 쓰는 사람들도 많고 좋은 기능들이 많아서 배워두면 좋다.

7. Fullpage.js : 웹페이지를 PPT처럼 버튼 누르면 다음 슬라이드 보여주고 그런 식으로 만들어준다.
