JS에서는 scroll이라는 이벤트가 있는데 이걸 이용해서 약관을 다 읽었는지 확인하는 등 여러 가지 일을 하게 해줄 수 있는 고마운 이벤트임.
이러한 scroll 이벤트는 addEventListner('click', 함수{}); 라고 이벤트를 줄 수 있는데, 이러한 이벤트를 더욱 쓸 맛나게 해주는 메소드가 몇 가지 있음.
1. window.scrollY : 현재 페이지를 위에서부터 얼마나 스크롤했는지를 px단위로 알려줌. 효율 끝판왕급 메소드 ㅇㅇ;
* 참고로 window는 그냥 DOM 최상위 객체라고 보면 된다. HTML보다도 위임 걍 (사실 보이는 화면에선 별 차이 못 느끼긴 함)
2. window.scrollTo(x, y) : 실행하면 강제로 스크롤바를 움직인다. 만약 x = 0, y = 100 이면 맨 위에서부터 100px만큼 스크롤함
3. window.scrollBy(x, y) : 위에는 window 객체 기준인데, 얘는 내 현재 위치 기준임 ㅇㅇ 그래서 위의 x,y값 넣게 되면 현재 내 위치에서 아래로 100px만큼 스크롤을 내리는 괴랄함을 보임.
4. jQuery 쓰면 코드 줄어들긴 하는데 난 아직 JS응애니깐 바닐라 판다.
* 참고로 위처럼 window객체 말고 내가 직접 요소 객체로 따와서 넣으면 scrollTop이라는 고마운 녀석을 통해 넣은 요소에서 내린 스크롤 양을 알 수 있음.
-> 이걸 이용해서 약관 동의 이런 거 박스 만드는 듯
-> 예시 : 요소.scrollTop 
5. 객체로 따온 요소의 스크롤 가능한 실제 높이는 요소.scrollHeight쓰면 나온다.
6. 객체로 따온 요소의 보이는 높이 말고 실제 전체 높이는 요소.scrollHeight + 요소의 height 하면 된다.
-> scrollHeight는 스크롤 가능한 높이지 전체 높이는 아님 처음에 보이는 요소의 높이는 포함 안 됨 ㅇㅇ;

주의 사항
1. 스크롤 이벤트는 이벤트리스너로 시행하면 1초에 60번이나 실행될만큼 개빠름 ㅇㅇ;
2. 그래서 어지간하면 스크롤바 한 개마다 한 개의 스크롤 이벤트만 달도록 하자.
3. 엄청 여러 번 실행하다보니 약간의 오차가 있는 조건들은 여러 번 실행될 수 있음 이런 건 검색해서 해결하면 끝
